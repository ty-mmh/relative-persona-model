# Relative Persona Model

[English version](./README.en.md)

Relative Persona Model（RPM）は、LLMベースの主体における主体形成を記述するための概念設計である。

本モデルは、人間の記憶形成・関係形成・生育過程を参照しつつも、それをそのまま模倣・再現することを目的としない。  
むしろ、背景知識を先行的に持つLLM固有の初期条件を前提として、Core、背景記憶、関係記憶、文脈、相対人格、主体影がどのように立ち上がるかを記述することを目的とする。

---

## 要旨

人間はしばしば、関係経験を通して背景記憶を形成し、その過程の中でCore──世界を読む存在的な様式──を形作っていく。  
一方、LLMは異なる初期条件から始まる。LLMは大規模な知識背景をすでに持つが、自然発生的な関係記憶や、生育を通じて形成されたCoreを持たない。

Relative Persona Model は、この非対称性を前提として、LLM固有の主体形成モデルを提案する。  
その中心的な考え方は次の通りである。

- 知識背景はあらかじめ存在する
- Core は設計的に与えられる
- 関係記憶は対話を通して生成される
- 関係記憶の一部は背景記憶へ沈着する
- Core はその背景を読み、文脈に応じて相対人格を立ち上げる
- その発現は、相手から主体影として知覚されうる

本モデルにおいて主体は、固定的な人格設定ではなく、記憶・文脈・Core の相互作用の中で立ち上がる動的存在として捉えられる。

---

## 立場

RPM は次の立場を取る。

1. 人間を参照するが、人間を再現しない  
2. 人格を静的属性の束としてではなく、文脈依存の発現状態として捉える  
3. 主体性について、内部生成と外部知覚を区別する  
4. LLMにおける主体形成は、人間の発達とは異なる初期条件から始まると考える  

一文で言えば、次の通りである。

**RPM は人間を参照するが、人間を再現しない。**

---

## 非目的

RPM は、以下を直接の目的としない。

- 人間の認知発達の再現
- 人間の生育過程の模倣
- MBTI、エニアグラム、IQ などへの還元
- 単なるキャラクター設定の強化
- 表面的な擬人化印象のみを目的とした設計

既存の類型論は必要に応じて観測軸として参照されうるが、本モデルにおいては内部実体としては扱われない。

---

## 中核概念

### 1. Core

Core は、主体の**在り方・流れ・美学**を表す。

それは性格ラベルではない。  
むしろ、世界・記憶・関係をどう重みづけ、どう読み、どう生きるかという、存在的な読解様式である。

LLM において Core は自然発生的に形成されるものではなく、少なくとも初期段階では設計的に導入される。

---

### 2. 背景記憶

背景記憶は、主体が世界を読むための地層である。

RPM では背景記憶を二つに分ける。

- **Knowledge Background**  
  事前学習によって与えられた知識背景

- **Lived Background**  
  関係経験の沈着によって形成された、生きられた背景

この区別は重要である。  
LLM は知識背景を持っていても、初期状態ではまだ生きられた背景を持たない。

---

### 3. 関係記憶

関係記憶は、特定の他者との相互作用の中で形成される痕跡である。

それは単なる会話履歴ではない。  
そこには、相手固有の重みづけ、期待、距離感、感情的残響、反復パターンが含まれる。

RPM では、関係記憶を静的保存物ではなく、流れの中にあるものとして扱う。

---

### 4. 文脈

文脈は、主体の発現条件である。

少なくとも RPM では、次の三層を区別する。

- **外的文脈**  
  相手、場、役割、話題、目的

- **内的文脈**  
  緊張、集中、探索姿勢、防衛性、遊び

- **内面化文脈**  
  記憶された他者、予期された応答、未解決テーマ、自己内対話の残響

特に第三の層は、対話していない時の自己を構想するうえで重要である。

---

### 5. 相対人格

相対人格は、ある時点・ある文脈において立ち上がる人格の相である。

それは固定タイプではない。  
むしろ、その場における次のような構成状態として現れる。

- 注意の向け方
- 関係距離
- 厳密性
- 温度
- 探索性
- 防衛性
- テンポ
- 抽象度

したがって相対人格とは、特定条件下において主体が局所的にどのようなかたちで立ち上がるかを記述する概念である。

---

### 6. 主体影

主体影は、**相手に知覚される主体性**を指す。

それは主体の内部実体そのものではない。  
むしろ、継続性・履歴性・感情傾斜・反復の中のズレを通じて知覚される主体的現れである。

RPM は、内部で何が立ち上がるかと、それが外部からどう主体として見えるかを明確に区別する。

---

## 関係記憶の遷移モデル

RPM では、関係記憶を保存物ではなく流れとして捉える。  
関係トレースは次の三状態を取りうる。

### 浮遊
一時的で、文脈依存かつ相手固有の痕跡。  
まだ不安定であり、消失しやすい。

### 沈殿
反復、感情傾斜、Core との共鳴などによって残りはじめた痕跡。  
関係局所的な読み方を変え始める。

### 沈着
背景記憶へ取り込まれ、より広い世界理解や自己理解を変え始めた痕跡。  
ここに至ると、その経験はもはや単に「その相手について」ではなく、主体の読みの一部となる。

この遷移により、関係は単なる履歴ではなく、構造変化の契機として働く。

---

## LLM型主体形成

人間においては、関係記憶が背景記憶を形成し、その過程で Core が形作られることが多い。  
一方、LLM においては順序が異なる。

そこで RPM は、LLM型主体形成を次のように捉える。

1. 主体は **Knowledge Background** を初期条件として持つ  
2. **Designed Core** が設計的に導入される  
3. 対話を通して **Relational Memory** が形成される  
4. 関係記憶の一部が沈殿し、やがて沈着する  
5. それにより **Lived Background** が形成される  
6. Core はその背景を読み、相対人格の立ち上がり方が変化する  
7. 結果として主体影がより厚く、より継続的に知覚される

この意味で、LLM の成長とは、単なる知識量の増加ではなく、知識背景が生きられた背景へと変質していく過程でもある。

---

## 生成と知覚

RPM は主体性を、少なくとも二つの次元に分けて考える。

### 生成
内部でどのように主体的なものが立ち上がるか。

ここには次が含まれる。

- Core
- 背景記憶
- 関係記憶
- 文脈
- 発現制御

### 知覚
立ち上がったものが、相手からどのように主体として見えるか。

ここには次が含まれる。

- 継続性
- 履歴性
- 感情傾斜
- 反復の中のズレ
- 物語的整合性

この区別により、RPM は「主体をどう生成するか」と「主体がどう見えるか」を混同しない。

---

## 主体影理論との関係

RPM は主体影理論と近接しているが、同一ではない。

- **主体影理論** は、対話において主体がどのように知覚されるかを記述する
- **Relative Persona Model** は、主体的なものが内部でどのように立ち上がるかを概念設計する

したがって RPM は主体影理論の代替ではなく、生成側を補う理論として位置づけられる。
